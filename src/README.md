# Frontend приложение для бронирования столиков в ресторане.

Пользователь проходит три шага:

1. Выбор даты, времени и количества гостей
2. Ввод контактной информации
3. Выбор зоны ресторана и стола

После завершения данные отправляются в API для создания брони.

# Использеумый стек технологий.

* Vue 3
* TypeScript
* Vite
* Axios
* Composition API

# Уставновка проекта.

1. Клонировать репозиторий:
git clone <repo-url>
cd restaurant-bookingVue

2. Установить зависимости:

npm install

3. Запустить проект:

npm run dev

Приложение будет доступно по адресу:
http://localhost:5173

# Логика шагов.

Step 1 — Дата и гости:

Пользователь выбирает:

* дату визита
* время визита
* количество гостей

Ограничения:

* нельзя выбрать дату в прошлом
* нельзя выбрать прошедшее время
* время работы ресторана: *10:00 — 23:00*
* количество гостей ограничено *maxCapacity*, получаемым из API

Endpoint:

GET /max-capacity

Step 2 — Контактные данные:

Пользователь вводит:

* имя
* номер телефона
* VIP статус

Валидация:

* имя — только буквы
* минимум 2 символа
* телефон форматируется автоматически:

+7 (999) 999-99-99

Step 3 — Выбор стола:

На этом шаге пользователь:

1. выбирает *placement (зону ресторана)*
2. выбирает *конкретный стол*

Данные загружаются из API:

GET /placements
GET /tables

После выбора выполняется отправка брони.

# API.

Получение максимальной вместимости:

GET /max-capacity

Ответ:
json
{
  "max_capacity": 20
}

Получение зон ресторана:

GET /placements

Получение столов:

GET /tables

Создание брони:

POST /reservations

Пример запроса:

```json
{
  "walkin_date": "2026-03-10",
  "walkin_time": "18:00",
  "no_of_persons": 4,
  "full_name": "Ivan Ivanov",
  "mobile": "+7 (999) 999-99-99",
  "placement_uuid": "placement-id",
  "table_uuid": "table-id",
  "timezone": "Europe/Moscow",
  "is_vip": false
}
```

# Особенности реализации

* валидация форм на каждом шаге
* динамическая загрузка данных из API
* форматирование телефона
* защита от выбора прошедшего времени
* ограничения времени работы ресторана
* поддержка timezone пользователя
